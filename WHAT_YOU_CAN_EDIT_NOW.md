## What Each Section Does

### 📄 Overview Section (NEW!)
**What it is:** The introductory text at the top of each country page

**What you can edit:**
- Left column text (general information about transfer pricing in the country)
- Right column text (additional details, regulations, etc.)

**Features:**
- Rich text editor (bold, italic, lists, links)
- Two-column layout on the public page
- Pre-filled with migrated content

**Example:**
```
Left Column:
"Qatar has established comprehensive transfer pricing 
regulations to ensure that transactions between related 
parties are conducted at arm's length..."

Right Column:
"The Qatar Tax Authority (QTA) requires taxpayers to 
maintain detailed documentation to support their 
transfer pricing positions..."
```

### 🛡️ Key Regulatory Frameworks (NEW!) - THE LAW SECTIONS
**What it is:** The 3 main law/regulation boxes on each country page

**What you can edit:**
- Framework 1: Title and description
- Framework 2: Title and description
- Framework 3: Title and description

**Features:**
- Rich text editor for descriptions
- Exactly 3 frameworks (matches the page design)
- Pre-filled with migrated content

**Example:**
```
Framework 1:
Title: "Transfer Pricing Regulations"
Description: "Qatar's transfer pricing rules are based 
on the OECD Transfer Pricing Guidelines..."

Framework 2:
Title: "Documentation Requirements"
Description: "Taxpayers must prepare and maintain 
master file, local file, and country-by-country 
reporting..."

Framework 3:
Title: "Penalties and Compliance"
Description: "Non-compliance with transfer pricing 
regulations may result in penalties of up to..."
```

### 📋 Documentation Cards (NEW!)
**What it is:** Expandable cards with detailed documentation

**What you can edit:**
- Add new cards
- Edit existing cards
- Remove cards
- Title, short description, and detailed content for each card

**Features:**
- Rich text editor for detailed content
- Add/remove cards dynamically
- Pre-filled with migrated content

**Example:**
```
Card 1:
Title: "Transfer Pricing Documentation"
Short Description: "Requirements for TP documentation"
Detailed Content: "Detailed explanation of what 
documents are required, deadlines, formats..."

Card 2:
Title: "Advance Pricing Agreements"
Short Description: "APA process in Qatar"
Detailed Content: "Step-by-step guide to applying 
for an APA, requirements, timeline..."
```

## WYSIWYG Editor Features

The rich text editor (TinyMCE) provides:

**Formatting:**
- **Bold** text
- *Italic* text
- Bullet lists
- Numbered lists
- Links
- Headings (H1-H6)

**Toolbar:**
```
[Undo] [Redo] | [Format ▼] | [B] [I] | [≡] [≡] [≡] [≡] | 
[• List] [1. List] [←] [→] | [🔗 Link] | [⌫ Clear] | [</>]
```

**Security:**
- Dangerous HTML (scripts, iframes) automatically removed
- Only safe formatting tags allowed
- Content sanitized before saving

## How to Edit Law Sections (Step-by-Step)

1. **Log in to admin panel**
   - Go to https://hexatp.com/admin/login.php
   - Username: `admin`, Password: `Admin123!`

2. **Go to Countries List**
   - Click "Countries List" in the navigation

3. **Click Edit on a country**
   - Find the country you want to edit (e.g., Qatar)
   - Click the "Edit" button

4. **Scroll down to "Key Regulatory Frameworks"**
   - This is where the law sections are!
   - You'll see 3 framework boxes

5. **Edit Framework 1 (or 2, or 3)**
   - Click in the Title field
   - Edit the title text
   - Click in the Description editor
   - Edit the description (use formatting toolbar)

6. **Save your changes**
   - Scroll to the bottom
   - Click "Save & Publish"
   - Wait for success message

7. **View on the public page**
   - Go to the country page (e.g., https://hexatp.com/qatar.html)
   - Your changes should be visible!

## Tips for Editing

✅ **DO:**
- Use the formatting toolbar for bold, italic, lists
- Keep titles concise (under 255 characters)
- Use bullet points for readability
- Add links to official sources
- Save frequently (Save as Draft)
- Preview changes on the public page

❌ **DON'T:**
- Paste formatted text from Word (use "Paste as Text")
- Use excessive formatting (keep it clean)
- Leave required fields empty
- Delete all 3 frameworks (keep at least the structure)
- Forget to click "Save & Publish"

## Troubleshooting

**Q: I don't see the new sections**
A: Clear your browser cache (Ctrl+Shift+R) and refresh

**Q: The editor doesn't load**
A: Check your internet connection (TinyMCE loads from CDN)

**Q: Changes don't save**
A: Check that you clicked "Save & Publish" and wait for confirmation

**Q: I see old content**
A: The migrated content is pre-filled - you can edit it!
